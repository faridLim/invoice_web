# 개인 개발 블로그 MVP PRD

## 핵심 정보

**목적**: Notion을 CMS로 활용하여 글 작성 즉시 블로그에 자동 반영되는 개인 기술 블로그 운영
**사용자**: 기술 글을 Notion으로 작성하고 별도 배포 없이 블로그를 운영하려는 개인 개발자

---

## 사용자 여정

```
1. 홈 페이지 (최근 글 목록)
   ↓ 카테고리 탭 클릭

2. 카테고리 필터 적용 (홈 페이지 내 필터)
   ↓ 검색창에 키워드 입력

   [검색 결과 있음] → 글 목록 갱신 → 글 카드 클릭
   [검색 결과 없음] → 빈 상태 안내 메시지 표시
   ↓

3. 글 상세 페이지
   ↓ 읽기 완료 후

4. 뒤로 가기 버튼 또는 헤더 로고 클릭 → 홈 페이지로 복귀
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|---------------|-------------|
| **F001** | Notion 글 목록 조회 | Notion API로 발행 상태 글만 필터링하여 가져오기 | 블로그의 핵심 콘텐츠 소스 | 홈 페이지 |
| **F002** | 글 상세 표시 | Notion 페이지 블록을 HTML로 렌더링 | 독자가 글을 읽는 핵심 기능 | 글 상세 페이지 |
| **F003** | 카테고리 필터링 | 선택한 카테고리에 해당하는 글만 표시 | 콘텐츠 탐색 핵심 경로 | 홈 페이지, 카테고리 페이지 |
| **F004** | 키워드 검색 | 제목 및 태그 기준으로 글 검색 | 원하는 글 빠르게 찾기 | 홈 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|---------------|-------------|
| **F010** | Notion API 연동 | Notion Integration 토큰 및 데이터베이스 ID 환경변수 관리 | 모든 기능의 데이터 소스 | 전체 (서버 공통) |
| **F011** | ISR 캐싱 | 글 목록과 상세 페이지를 일정 주기로 재검증 | Notion API 요청 수 최소화 | 홈 페이지, 글 상세 페이지 |

### 3. MVP 이후 기능 (제외)

- 댓글 기능
- 뉴스레터 구독
- 좋아요 / 조회수 카운터
- 목차(TOC) 자동 생성
- 관련 글 추천
- 소셜 공유 버튼
- 다크모드 토글 (Tailwind 기본 다크모드 설정으로 대체)
- RSS 피드

## 메뉴 구조

```
개인 개발 블로그 내비게이션
├── 로고/블로그명 (홈으로 이동)
│   └── 기능: F001 (글 목록 초기화 및 홈으로 복귀)
├── 홈
│   └── 기능: F001, F004 (최근 글 목록, 키워드 검색)
└── 카테고리
    └── 기능: F003 (카테고리별 글 목록)

공통 레이아웃 (모든 페이지)
├── 헤더: 블로그명, 홈 링크, 카테고리 링크
└── 푸터: 블로그 소개 한 줄
```

## 페이지별 상세 기능

### 홈 페이지

> **구현 기능:** `F001`, `F003`, `F004`, `F011` | **인증:** 불필요

| 항목 | 내용 |
|------|------|
| **역할** | 블로그 진입점. 발행된 전체 글 목록을 최신순으로 표시하고 탐색 진입 |
| **진입 경로** | 직접 URL 접근, 헤더 로고 클릭, 헤더 홈 메뉴 클릭 |
| **사용자 행동** | 최근 글 카드 탐색, 카테고리 탭 클릭으로 필터 적용, 검색창에 키워드 입력, 글 카드 클릭으로 상세 이동 |
| **주요 기능** | - Notion API에서 Status=발행됨 글 목록 가져오기 (F001)<br>- 카테고리 탭 UI로 필터링 (F003)<br>- 제목/태그 키워드 검색 (F004)<br>- ISR로 60초 주기 재검증 (F011)<br>- **글 카드 클릭** 시 글 상세 페이지로 이동 |
| **다음 이동** | 글 카드 클릭 → 글 상세 페이지 |

---

### 글 상세 페이지

> **구현 기능:** `F002`, `F011` | **인증:** 불필요

| 항목 | 내용 |
|------|------|
| **역할** | Notion 페이지 본문을 렌더링하여 독자에게 글 전체 내용 제공 |
| **진입 경로** | 홈 페이지 또는 카테고리 페이지에서 글 카드 클릭 |
| **사용자 행동** | 제목, 카테고리, 태그, 발행일 확인 후 본문 읽기, 뒤로 가기로 목록 복귀 |
| **주요 기능** | - Notion 페이지 블록 파싱 및 HTML 렌더링 (F002)<br>- 제목, 카테고리, 태그, 발행일 메타 표시<br>- ISR로 재검증 (F011)<br>- **뒤로 가기** 링크 (홈 페이지 또는 카테고리 페이지로) |
| **다음 이동** | 뒤로 가기 → 홈 페이지 또는 카테고리 페이지 |

## 데이터 모델

> Notion 데이터베이스가 원천 데이터. 앱 내 DB 없이 Notion API 응답을 타입으로 정의.

### Post (Notion 페이지 매핑 타입)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | Notion 페이지 ID | string (UUID) |
| title | 글 제목 | string |
| category | 카테고리 (단일 선택) | string |
| tags | 태그 목록 | string[] |
| publishedAt | 발행일 | Date |
| status | 초안 / 발행됨 | "draft" / "published" |
| slug | URL용 식별자 (id 기반) | string |

### NotionBlock (본문 블록 타입)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 블록 ID | string |
| type | 블록 종류 (paragraph, heading 등) | string |
| content | 텍스트 또는 하위 블록 내용 | object |
| parentId | 부모 블록 ID | → NotionBlock.id |

## 기술 스택

### 프론트엔드 프레임워크

- **Next.js 16** (App Router) - 프로젝트 기본 프레임워크, ISR 지원
- **TypeScript 5.6+** - 타입 안전성
- **React 19** - UI 렌더링

### 스타일링 & UI

- **Tailwind CSS v4** - 유틸리티 CSS (설정 파일 없는 새 엔진)
- **shadcn/ui** (radix-nova style) - 카드, 배지, 탭 컴포넌트
- **Lucide React** - 아이콘

### Notion 연동

- **@notionhq/client** - 공식 Notion SDK (글 목록, 블록 조회)
- **notion-to-md** - Notion 블록을 Markdown으로 변환
- **react-markdown** + **rehype-highlight** - Markdown 렌더링 및 코드 하이라이팅

### 배포 & 환경

- **Vercel** - Next.js 최적화 배포, ISR 지원
- **환경변수**: `NOTION_TOKEN`, `NOTION_DATABASE_ID`

### 패키지 관리

- **npm** - 의존성 관리
